Sobre esta Aplicação

1. Arquitetura MVT

Esta aplicação utiliza o padrão Model-View-Template do Django:

  • Model: Define a estrutura dos dados (ex: Projetos, Tecnologias).
  • View: Gere a lógica da aplicação e processa pedidos.
  • Template: Responsável pela apresentação visual.

O utilizador faz um pedido → a View processa → consulta o Model → devolve um Template com dados.

2. Modelação

A aplicação foi modelada com várias entidades como Projeto, Tecnologia, Unidade Curricular e TFC.

  • Projeto → Tecnologia (ManyToMany)
  • Projeto → Unidade Curricular (ForeignKey)
  • TFC → Curso (ForeignKey)
